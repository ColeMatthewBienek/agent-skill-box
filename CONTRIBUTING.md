# Contributing

Thanks for helping improve Agent Skill Box.

## Before opening a pull request

1. Open an issue first for changes that add a skill, significantly change a skill's behavior, or alter installation instructions. Small documentation corrections and focused fixes can go straight to a pull request.
2. Fork the repository and create a branch from `main`.
3. Keep each pull request focused on one improvement.
4. Preserve the Agent Skills format: every skill directory needs a `SKILL.md` with valid frontmatter and any referenced assets must be included.
5. Update the relevant README when installation, invocation, or behavior changes.

## Pull request expectations

- Explain the problem and the intended outcome.
- Include a concise test or validation note. For documentation-only changes, say so.
- Do not include unrelated formatting changes or generated files.
- Be respectful and follow the [Code of Conduct](CODE_OF_CONDUCT.md).

Maintainers will review pull requests for correctness, clarity, scope, and compatibility with the supported agent runtimes.
